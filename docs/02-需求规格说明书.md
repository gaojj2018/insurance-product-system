# 保险核心系统需求规格说明书

## 一、项目概述

### 1.1 项目名称
保险核心系统

### 1.2 项目目标
构建一套完整的保险业务核心系统，支持产品管理、投保、核保、保单管理、理赔、财务等核心业务流程。

### 1.3 系统范围
- 个人人寿保险
- 意外伤害保险
- 健康保险
- 财产保险

---

## 二、功能需求

### 2.1 产品管理

#### 2.1.1 产品配置
- 产品基本信息维护（代码、名称、类型）
- 保障期间配置
- 缴费期间配置
- 保额范围设置

#### 2.1.2 条款配置
- 基本条款管理
- 附加条款管理
- 条款内容维护

#### 2.1.3 险种配置
- 主险配置
- 附加险配置
- 保额/保费设置

### 2.2 客户管理

#### 2.2.1 客户信息
- 客户基本信息（姓名、证件类型、证件号码）
- 联系方式（手机、邮箱、地址）
- 客户类型（个人/企业）
- 风险等级

#### 2.2.2 客户状态
- 正常状态
- 冻结状态

### 2.3 投保管理

#### 2.3.1 投保申请
- 选择产品
- 填写投保信息
- 选择险种
- 计算保费
- 提交投保

#### 2.3.2 投保状态
- 待核保
- 核保中
- 核保通过
- 核保拒绝
- 已撤单

### 2.4 核保管理

#### 2.4.1 核保审核
- 查看投保信息
- 审核决定
- 填写核保意见

#### 2.4.2 核保结果
- 通过
- 拒绝
- 延期
- 条件承保

### 2.5 保单管理

#### 2.5.1 保单生成
- 自动生成保单号
- 保单起止日期
- 保单状态

#### 2.5.2 保单状态
- 生效
- 失效
- 终止
- 退保

### 2.6 理赔管理

#### 2.6.1 理赔申请
- 报案登记
- 提交理赔材料

#### 2.6.2 理赔审核
- 材料审核
- 理算处理
- 理赔决定

#### 2.6.3 理赔状态
- 待处理
- 处理中
- 理算中
- 已支付
- 拒绝赔付

### 2.7 财务管理

#### 2.7.1 保费管理
- 保费确认
- 保费退还

#### 2.7.2 理赔支付
- 理赔金额确认
- 支付处理

#### 2.7.3 报表统计
- 保费收入统计
- 理赔支出统计

---

## 三、非功能需求

### 3.1 性能要求
- 页面响应时间 < 3秒
- API响应时间 < 1秒

### 3.2 可用性要求
- 系统可用率 > 99.9%
- 支持7x24小时运行

### 3.3 安全性要求
- 用户认证与授权
- 数据加密传输
- 操作日志记录

---

## 四、用户角色

| 角色 | 说明 |
|------|------|
| ADMIN | 系统管理员 |
| AGENT | 保险代理人 |
| UNDERWRITER | 核保员 |
| CLAIMS_OFFICER | 理赔员 |
| CUSTOMER | 客户 |

## 五、接口清单

### 5.1 认证接口
- POST /api/auth/login - 用户登录
- POST /api/auth/logout - 用户登出

### 5.2 产品接口
- POST /api/product/page - 分页查询产品
- GET /api/product/{id} - 获取产品详情
- POST /api/product - 创建产品
- PUT /api/product - 更新产品
- DELETE /api/product/{id} - 删除产品
- POST /api/product/publish/{id} - 发布产品
- POST /api/product/stop/{id} - 停售产品

### 5.3 条款接口
- GET /api/clause/product/{productId} - 获取产品条款
- POST /api/clause - 创建条款
- PUT /api/clause - 更新条款
- DELETE /api/clause/{id} - 删除条款

### 5.4 险种接口
- GET /api/coverage/product/{productId} - 获取产品险种
- POST /api/coverage - 创建险种
- PUT /api/coverage - 更新险种
- DELETE /api/coverage/{id} - 删除险种

### 5.5 客户接口
- GET /api/customer/page - 分页查询客户
- GET /api/customer/{id} - 获取客户详情
- POST /api/customer - 创建客户
- PUT /api/customer/{id} - 更新客户
- POST /api/customer/freeze/{id} - 冻结客户
- POST /api/customer/unfreeze/{id} - 解冻客户

### 5.6 投保接口
- POST /api/application/page - 分页查询投保
- GET /api/application/{id} - 获取投保详情
- POST /api/application - 提交投保
- POST /api/application/{id}/cancel - 撤单

### 5.7 核保接口
- POST /api/underwriting/page - 分页查询核保
- GET /api/underwriting/{id} - 获取核保详情
- POST /api/underwriting/{id}/approve - 核保通过
- POST /api/underwriting/{id}/reject - 核保拒绝

### 5.8 保单接口
- POST /api/policy/page - 分页查询保单
- GET /api/policy/{id} - 获取保单详情

### 5.9 理赔接口
- POST /api/claims/page - 分页查询理赔
- GET /api/claims/{id} - 获取理赔详情
- POST /api/claims - 提交理赔
- POST /api/claims/{id}/process - 处理理赔
- POST /api/claims/{id}/pay - 理赔支付

### 5.10 财务接口
- POST /api/finance/premium/page - 保费列表
- POST /api/finance/premium/confirm - 确认保费
- POST /api/finance/premium/refund - 退还保费
- GET /api/finance/stats - 财务统计
- GET /api/finance/report - 财务报表
