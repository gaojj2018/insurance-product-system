### 1.2 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | Vue3 + Element Plus + Vite |
| 网关 | Spring Cloud Gateway |
| 注册/配置 | Eureka + Spring Cloud Config |
| 服务框架 | Spring Boot 3.2.0 |
| Java版本 | Java 17 |
| 数据库 | MySQL 8.0 |
| 缓存 | Redis |
| ORM | MyBatis Plus |
| 安全 | JWT |

## 三、业务流程

### 3.1 投保流程
```
客户 → 投保申请 → 产品选择 → 填写信息 → 提交投保 → 
核保审核 → (通过) → 生成保单 → (拒绝) → 投保失败
```

### 3.2 理赔流程
```
客户报案 → 理赔申请 → 材料审核 → 理算 → 审批 → 支付
```

## 四、API设计

### 4.1 认证接口
- POST /api/auth/login - 登录
- POST /api/auth/logout - 登出

### 4.2 产品接口
- POST /api/product/page - 产品列表
- GET /api/product/{id} - 产品详情
- POST /api/product - 创建产品
- PUT /api/product - 更新产品

### 4.3 客户接口
- GET /api/customer/page - 客户列表
- GET /api/customer/{id} - 客户详情
- POST /api/customer - 创建客户

### 4.4 投保接口
- POST /api/application/page - 投保列表
- POST /api/application - 提交投保
- GET /api/application/{id} - 投保详情

### 4.5 核保接口
- POST /api/underwriting/page - 核保列表
- POST /api/underwriting/{id}/approve - 核保通过
- POST /api/underwriting/{id}/reject - 核保拒绝

### 4.6 保单接口
- POST /api/policy/page - 保单列表
- GET /api/policy/{id} - 保单详情

### 4.7 理赔接口
- POST /api/claims/page - 理赔列表
- POST /api/claims - 提交理赔
- POST /api/claims/{id}/process - 处理理赔
- POST /api/claims/{id}/pay - 理赔支付

## 五、安全设计

### 5.1 认证
- JWT Token认证
- Token有效期: 24小时

### 5.2 授权
- 基于角色的访问控制 (RBAC)
- 角色: ADMIN, AGENT, UNDERWRITER, CLAIMS_OFFICER, CUSTOMER
