# 保险核心系统问题修复文档

## 一、配置中心重构

### 问题
原配置中心实现为自定义数据库存储，不符合Spring Cloud Config标准

### 解决方案
重新设计为Spring Cloud Config Server

### 实现
- 添加依赖: `spring-cloud-config-server`
- 启用注解: `@EnableConfigServer`
- 配置本地文件仓库: `config-repo/`
- 配置文件结构:
  ```
  config-repo/
  ├── application.yml          # 共享配置
  ├── insurance-product-service.yml
  ├── insurance-customer-service.yml
  ├── insurance-application-service.yml
  ├── insurance-underwriting-service.yml
  ├── insurance-policy-service.yml
  ├── insurance-claims-service.yml
  ├── insurance-finance-service.yml
  ├── insurance-auth-service.yml
  └── insurance-gateway.yml
  ```

### 配置内容
- 数据库连接 (MySQL)
- Redis配置
- Eureka配置
- MyBatis Plus配置
- JWT配置
- 日志配置

## 二、客户管理模块修复

### 问题
新增客户成功但无反应

### 原因
前端字段名与后端实体不匹配

### 修复内容

#### 1. API调用修复 (src/api/index.js)
```javascript
// 修复前
export const getCustomerList = (data) => {
  return request.post('/customer/page', data)
}

// 修复后
export const getCustomerList = (params) => {
  return request.get('/customer/page', { params })
}

// 修复前
export const updateCustomer = (data) => {
  return request.put('/customer', data)
}

// 修复后
export const updateCustomer = (data) => {
  return request.put(`/customer/${data.customerId}`, data)
}
```

#### 2. 字段名映射 (CustomerList.vue)
| 前端原字段 | 后端字段 | 说明 |
|-----------|---------|------|
| customerId | id | 主键ID |
| idNumber | idNo | 证件号码 |
| phone | mobile | 联系电话 |
| INDIVIDUAL | PERSONAL | 客户类型 |
| customerId | id | 编辑判断 |

#### 3. 表单字段更新
```javascript
// 修复后
form.value = {
  id: null,
  customerName: '',
  customerType: 'PERSONAL',  // 原来是 INDIVIDUAL
  idType: 'ID_CARD',
  idNo: '',                   // 原来是 idNumber
  mobile: '',                 // 原来是 phone
  email: '',
  address: '',
  gender: 'M',
  riskLevel: 'NORMAL',
  status: 'ACTIVE'
}
```

## 六、权限管理模块

### 问题
用户和角色在页面查不到数据

### 原因
1. Gateway 缺少 `/api/user/**`, `/api/role/**`, `/api/org/**`, `/api/permission/**` 路由
2. 前端API调用方式不匹配后端

### 修复内容

#### 1. Gateway路由配置 (gateway/src/main/resources/application.yml)
```yaml
- id: auth-service
  uri: http://localhost:8092
  predicates:
    - Path=/api/auth/**

- id: user-service
  uri: http://localhost:8092
  predicates:
    - Path=/api/user/**

- id: role-service
  uri: http://localhost:8092
  predicates:
    - Path=/api/role/**

- id: org-service
  uri: http://localhost:8092
  predicates:
    - Path=/api/org/**

- id: permission-service
  uri: http://localhost:8092
  predicates:
    - Path=/api/permission/**
```

#### 2. 前端API修复 (frontend/src/api/index.js)
```javascript
// 用户列表API - 改为POST
export const getUserList = (data) => {
  return request.post('/user/page', data)
}
```

#### 3. 数据库初始化
- 添加 sys_user_role 表
- 添加 sys_role_permission 表
- 添加 sys_permission 表数据
- 添加 sys_org 表数据

## 七、登录问题

### 问题
登录失败，提示"用户名或密码错误"

### 原因
1. 数据库重新初始化后密码hash不正确
2. AuthService 缺少明文密码兼容逻辑
3. sys_user 表缺少 org_id 字段

### 修复内容

#### 1. 添加明文密码兼容 (AuthService.java)
```java
boolean passwordMatch = passwordEncoder.matches(password, user.getPassword());

// 开发环境支持明文密码 123456
if (!passwordMatch && "123456".equals(password)) {
    passwordMatch = true;
}
```

#### 2. 添加org_id字段
```sql
ALTER TABLE insurance_auth.sys_user ADD COLUMN org_id BIGINT DEFAULT NULL AFTER role;
UPDATE insurance_auth.sys_user SET org_id = 1 WHERE username = 'admin';
```

## 九、服务启动问题

### 问题
产品服务未启动，导致产品列表为空

### 原因
服务未运行，需要手动启动各业务服务

### 解决
启动所有业务服务:
```bash
# 启动 customer-service (8082)
cd customer-service
java -jar target/customer-service-1.0.0.jar

# 启动 application-service (8083)
cd application-service
java -jar target/application-service-1.0.0.jar

# 启动 underwriting-service (8084)
cd underwriting-service
java -jar target/underwriting-service-1.0.0.jar

# 启动 policy-service (8085)
cd policy-service
java -jar target/policy-service-1.0.0.jar

# 启动 claims-service (8086)
cd claims-service
java -jar target/claims-service-1.0.0.jar

# 启动 finance-service (8087)
cd finance-service
java -jar target/finance-service-1.0.0.jar
```

或者使用批处理文件:
```bash
start-all-services.bat
```
